# PlusGrade Data Scientist Technical Assignment - by Emulie Chhor

## Usage

```
# installing libraries
pip install uv
uv sync

# to run the project
uv run python src/main.py

# to see experiment results
uv run mlflow ui # go to http://localhost:5000
```

## Code Structure

**Important Files**

- `src/config.py` => Configuration files to be used during training. Can redifine offer coverage floor (constraint 2)
- `src/main.py` => Training the 2 predictives models and offer allocation strategies


**Directory**

```
├── assignments =>
├── mlruns => saved mlflow artifacts when training models
├── playground => jupyter notebook with EDA, feature engineering and modeling ideation
├── prompts => prompts used to generate code
├── reports => write-ups
└── src
    ├── allocation => code for offer allocation and business strategies
    ├── data => utils to split data
    ├── features => logic for feature engineering
    ├── models => sklearn models for M1 and M2 using registry pattern
    └── utils => mlflow utils
```

**Notebooks**

```
playground
├── auto_EDA.ipynb => EDA generated by AI. Used to get a quick grasp of correlated features, confounding variables and potential feature transformations.
├── EDA.ipynb => Manual EDA for more specific questions that I have
├── M2_points_purchased.ipynb => Modeling Points Purchased in notebook for debugging purposes
├── M2_two_towers.ipynb => Attempting to improve M2 through two towers approach ie using probabilistic blending to improve prediction around tails
└── review_predictions.ipynb

```
